# 闲鱼订单使用与排障指南

本功能包含两条主线：
- 用户侧 `/redeem/xianyu`：输入邮箱 + 闲鱼订单号，自助核销。
- 管理侧 `/admin/xianyu-orders`：维护 Cookie、同步订单、手动核销/绑定兑换码。

## 1) 前置条件

### 1.1 功能开关

- 管理后台进入 `/admin/settings`，开启「闲鱼（订单同步/兑换）」。
- 未开启时，闲鱼相关页面/接口会返回 `403`（`FEATURE_DISABLED`）。

### 1.2 管理权限

- 管理页面 `/admin/xianyu-orders` 需要菜单权限 `xianyu_orders`。

### 1.3 闲鱼 Cookie（必须）

- 在 `/admin/xianyu-orders` 点击「凭证」，粘贴闲鱼（`goofish.com`）Cookie。
- Cookie 必须可用，且建议包含 `_m_h5_tk`，否则无法调用闲鱼订单详情接口。
- 系统在请求过程中可能刷新 Cookie，并自动写回配置。

### 1.4 兑换码库存（必须）

- 在 `/admin/redemption-codes` 准备渠道为 `xianyu` 的库存。
- 若创建 `warranty` 类型兑换码，必须填写 `serviceDays`（1-3650）。

## 2) 用户侧兑换流程

1. 用户访问 `/redeem/xianyu`。
2. 输入邮箱和订单号（订单号仅支持 6-30 位纯数字）。
3. 系统先调用 `check-order` 查本地订单。
4. 若本地不存在，再调用 `search-order` 用 Cookie 拉取闲鱼订单并入库。
5. 调用 `redeem-order` 自动选码并发起邀请。

成功后返回邀请结果（账号邮箱、当前人数、邀请状态）。

## 3) 后台运营流程

### 3.1 手动同步订单

- 入口：`/admin/xianyu-orders` -> 「同步订单」。
- 输入订单号后触发服务端同步。
- 若订单已存在，会返回“订单已同步（synced: false）”。

### 3.2 后台核销订单

- 在订单列表点击「核销」。
- 已核销订单不可重复核销。
- 订单状态包含“关闭”时不可核销（直接拒绝）。

### 3.3 绑定已核销兑换码

用于“兑换动作已完成、但订单还没绑定兑换码”的人工补录场景：
- 入口：`/admin/xianyu-orders` -> 「绑定码」。
- 仅允许绑定“已被使用”的兑换码。
- 同一个兑换码不能重复绑定到其他小红书/闲鱼订单。

## 4) 库存分配规则

默认策略为“严格今日库存”（`strictToday` 默认开启）：
- 优先找 `xianyu` 渠道、今日创建、未使用且未预留的兑换码。

当 `strictToday=false` 时，才会启用历史兜底：
- `00:00-07:59` 可尝试昨天库存。
- 之后可尝试任意日期库存。
- 使用历史库存时，要求账号过期时间至少大于当前时间 + 30 天。

当 `xianyu` 渠道配置允许 `allowCommonFallback` 时：
- 会按同样规则再尝试 `common` 渠道库存。

## 5) 质保与服务天数规则（闭环）

当前规则是“创建即真值”：
- 兑换阶段不接受 `orderType` 参数。
- 兑换不会覆盖兑换码创建时的 `order_type/service_days`。
- `warranty` 必须在创建时提供合法 `serviceDays`（1-3650）。

## 6) 常见错误与处理

- `请先在管理后台配置 Cookie`
  - 处理：去 `/admin/xianyu-orders` 保存有效 Cookie。
- `Cookie 缺少 _m_h5_tk，请重新登录闲鱼后复制 Cookie`
  - 处理：重新登录闲鱼后复制最新 Cookie。
- `该订单已关闭，无法进行核销`
  - 处理：关闭订单不可核销，只能更换有效订单。
- `xianyu_codes_not_configured`
  - 处理：未配置 `xianyu` 渠道库存。
- `xianyu_no_today_codes`
  - 处理：今日无 `xianyu` 码，需补当日库存（或改为非严格今日策略）。
- `xianyu_today_codes_exhausted`
  - 处理：今日库存已耗尽，补码后重试。
- `orderType 参数已弃用`
  - 处理：调用兑换接口时移除 `orderType/order_type` 字段。

## 7) 接口速查

用户侧（无需后台登录）：
- `POST /api/redemption-codes/xianyu/check-order`
- `POST /api/redemption-codes/xianyu/search-order`
- `POST /api/redemption-codes/xianyu/redeem-order`

管理侧（需要登录 + `xianyu_orders` 菜单权限）：
- `GET /api/xianyu/status`
- `POST /api/xianyu/config`
- `POST /api/xianyu/sync`
- `GET /api/xianyu/orders`
- `POST /api/xianyu/orders/:id/bind-code`
- `POST /api/xianyu/refresh-login`

## 8) 相关环境变量

- `XIANYU_LOGIN_REFRESH_ENABLED`：是否启用 Cookie 自动续期（默认 `true`）。
- `XIANYU_LOGIN_REFRESH_INTERVAL_MINUTES`：自动续期间隔分钟（默认 `30`）。
- `REDEEM_ORDER_STRICT_TODAY_DEFAULT`：订单核销是否默认严格今日库存（默认 `true`）。
