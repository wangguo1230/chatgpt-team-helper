# 各渠道兑换说明指南

本文档详细说明了本项目支持的各种兑换渠道、用户兑换流程及相应的管理配置。

---

## 1. 渠道概览

下表列出了系统中内置的主要渠道及其核心机制：

| 渠道标识 (Key) | 渠道名称 | 兑换凭证 | 用户操作入口 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| `common` | 通用 / 官网 | 兑换码 (XXXX-XXXX-XXXX) | `/redeem/common` | 标准 12 位分段式兑换码。 |
| `xhs` | 小红书 (XHS) | 小红书订单号 (P开头) | `/redeem/xhs` | 需管理员在后台配置小红书 Cookie 凭证，系统自动同步订单。 |
| `xianyu` | 闲鱼 | 闲鱼订单号 (纯数字) | `/redeem/xianyu` | 需管理员在后台配置闲鱼 Cookie，系统自动拉取订单详情。 |
| `linux-do` | LinuxDo 积分 | 无 (OAuth 授权支付) | `/redeem/open-accounts` | 使用 LinuxDo 平台的 Credit (LDC) 积分直接购买上车。 |
| `paypal` | PayPal | 兑换码 (XXXX-XXXX-XXXX) | `/redeem/common` | 针对 PayPal 支付场景分发的兑换码。 |
| `payment` | 站内支付 (ZPAY) | 无 (在线支付) | `/purchase` | 系统内置商品列表，支持多种扫码支付，完成后自动发放。 |

## 2. 详细兑换流程

### 2.1 兑换码模式 (Common / PayPal)
- **获取方式**：用户通过发卡网或其他人工渠道获得 12 位兑换码（如 `ABCD-1234-EFGH`）。
- **兑换流程**：
  1. 访问 `/redeem/common`。
  2. 输入用于接收邀请的 ChatGPT 账号邮箱。
  3. 输入兑换码并提交。
- **管理配置**：管理员需在 `/admin/redemption-codes` 预先生成对应渠道的兑换码。

### 2.2 订单同步模式 (小红书 / 闲鱼)
- **核心机制**：用户在对应平台下单后，无需等待管理员发码，通过订单号直接兑换。
- **兑换流程**：
  1. 访问 `/redeem/xhs` 或 `/redeem/xianyu`。
  2. 输入邮箱和订单号。
  3. 系统会自动连接小红书/闲鱼后台搜索该订单。
  4. 验证订单金额与状态（必须是已付款、未兑换过）。
  5. 匹配后端预留的对应渠道库存（兑换码），完成发放。
- **管理配置**：
  - 管理员必须在 `/admin/xhs-orders` 或 `/admin/xianyu-orders` 维持 Cookie/凭证的有效性。
  - 必须在 `/admin/redemption-codes` 准备好渠道为 `xhs` 或 `xianyu` 的兑换码**库存**。

### 2.3 授权支付模式 (LinuxDo 积分)
- **核心机制**：深度集成 LinuxDo API，消耗外部平台积分。
- **流程**：
  1. 访问 `/redeem/open-accounts`。
  2. 选定目标账号，点击“上车”。
  3. 系统调起 LinuxDo Credit 支付网关。
  4. 用户在 LinuxDo 授权扣分。
  5. 积分扣除成功后，后端自动核销对应的 `linux-do` 渠道库存并发送邀请。

### 2.4 站内购买模式 (ZPAY)
- **流程**：
  1. 访问 `/purchase`。
  2. 选择商品，提交订单。
  3. 在站内跳转支付页完成付款。
  4. 支付完成后系统自动跳转至结果页并完成账号授权。

## 3. 特殊兑换：掉号补录 (Recovery)

- **入口**：`/redeem/account-recovery`。
- **适用场景**：用户已加入的账号突然无法访问（封号或过期）。
- **逻辑**：
  - 用户输入邮箱。
  - 系统检索该邮箱在 30 天（可配置）内的有效订单。
  - 实时检测原账号是否封禁或访问异常。
  - 若异常，系统自动从补录池选一个新账号并重新发送邀请。
- **注意**：无质保订单不支持补录。

## 4. 管理员常见问题 (FAQ)

#### Q: 用户输入闲鱼订单号提示“暂无库存”？
**A**: 请去「兑换码管理」页面，点击生成兑换码，并将 **渠道 (Channel)** 设置为 `xianyu`。订单同步模式本质上是“自动消耗指定渠道的闲置兑换码”。

#### Q: 小红书订单一直同步失败？
**A**: 请去「小红书订单」页面检查凭证 (cURL) 是否过期，通常建议重新复制最新的 cURL 命令粘贴。

#### Q: 怎么控制不同渠道的质保时间？
**A**: 订单同步时系统会根据订单实收金额自动判断。例如闲鱼 5 元订单标记为 `no_warranty`，15 元订单标记为 `warranty`。
