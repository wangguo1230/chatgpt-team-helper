# 开放账号（前台标题：栗子LDC，含 Credit 订单）功能启用指南

本功能包含：
- 用户侧「开放账号页」：`/redeem/open-accounts`（基于 Linux DO OAuth）
- Credit 订单相关接口与后台管理：`/admin/credit-orders`

说明：
- 前台页面主标题当前展示为「栗子LDC」。
- 默认不再启用固定时段停兑（如 00:00-08:00）。

> 约定：**Credit 订单归属于开放账号功能**（禁用开放账号即禁用 Credit 订单相关能力）。

## 1) 开启功能开关

- 进入管理后台：`/admin/settings`
- 在「功能开关」中启用「开放账号（含 Credit 订单）」

禁用时相关页面与接口会返回 `403`（`FEATURE_DISABLED`）。

## 2) 配置 Linux DO OAuth

推荐在管理后台配置并入库：
- 进入：`/admin/settings` →「Linux DO OAuth 配置」
- 填写：
  - `LINUXDO_CLIENT_ID`
  - `LINUXDO_CLIENT_SECRET`
  - `LINUXDO_REDIRECT_URI`

## 3) 配置 Linux DO Credit（积分体系）

推荐在管理后台配置并入库：
- 进入：`/admin/settings` →「Linux DO Credit 配置」
- 填写：
  - `LINUXDO_CREDIT_PID`
  - `LINUXDO_CREDIT_KEY`

可选环境变量（见 `backend/.env.example`）：
- `LINUXDO_CREDIT_BASE_URL`（默认 `https://credit.linux.do/epay`）

## 4) 积分商品与库存配置

### 4.1. 积分价格与规则配置（环境变量）

修改后端 `backend/.env`（可参考 `backend/.env.example`），修改后需 **重启后端服务**：
```bash
# 上车所需要消耗的 Linux DO 积分（LDC），默认 30
OPEN_ACCOUNTS_CREDIT_COST=30

# 积分扣除订单显示的商品名称
OPEN_ACCOUNTS_CREDIT_TITLE=开放账号上车

# 购买后获得的上车有效期（天），默认 30
OPEN_ACCOUNTS_SUBSCRIPTION_DAYS=30

# 每日允许上车的总人数上限（防并发/刷单），默认 10。0表示不限制
OPEN_ACCOUNTS_DAILY_BOARD_LIMIT=0

# 每个用户每日上车次数限制（默认关闭；0 表示不限制）
OPEN_ACCOUNTS_USER_DAILY_BOARD_LIMIT_ENABLED=false
OPEN_ACCOUNTS_USER_DAILY_BOARD_LIMIT=0

# 订单支付有效期（等待用户授权扣积分的时间，分钟），默认 15
CREDIT_ORDER_EXPIRE_MINUTES=15
```

### 4.2. 库存（名额）补充机制

前端开放账号页展现的剩余名额不是一个独立数字，而是由**底层未使用的兑换码库存数量**决定的：

**增加库存步骤：**
1. 进入管理后台：`/admin/redemption-codes`（兑换码管理）
2. 点击 **生成兑换码**：
   - 渠道 **必须** 选择：`linux-do`
   - 绑定账号 **必须** 选择：你要补充库存的那个 ChatGPT 账号邮箱
   - 生成数量：你要补几份库存就填几个
3. 生成完成后，前台对应账号展示的可用名额即可增加。

> **注意：**只有在 `OPEN_ACCOUNTS_DAILY_BOARD_LIMIT > 0` 时，才会触发每日总计上车上限限制。

## 5) 可选：维护模式与其他业务参数

维护模式（环境变量）：
- `OPEN_ACCOUNTS_ENABLED`：默认 `true`（关闭会提示维护中）
- `OPEN_ACCOUNTS_MAINTENANCE_MESSAGE`

更多底层业务参数（见 `backend/.env.example`）：
- `CREDIT_ORDER_QUERY_MIN_INTERVAL_MS`
- `CREDIT_GATEWAY_SERVER_*`（是否允许服务端查单/退款等）

### 5.1 开放账号扫描（超员 + 安全）

开放账号扫描器现在包含两类检查，并统一输出一份扫描报告邮件：

1. **超员检查**：当账号人数超过开放账号容量上限（`OPEN_ACCOUNTS_CAPACITY_LIMIT` / 系统配置 `open_accounts_capacity_limit`）时，自动踢出超额成员并回写人数。
2. **安全检查**：当检测到账号访问异常（例如 token 失效、上游返回不可用）时，自动封号下架并清理该账号未使用兑换码。

可配置项（环境变量）：
- `OPEN_ACCOUNTS_SWEEPER_ENABLED`：是否启用扫描器。
- `OPEN_ACCOUNTS_SWEEPER_INTERVAL_MINUTES`：扫描间隔（分钟，优先级高于 `OPEN_ACCOUNTS_SWEEPER_INTERVAL_HOURS`）。
- `OPEN_ACCOUNTS_SWEEPER_INTERVAL_HOURS`：按小时间隔扫描（兼容旧配置）。
- `OPEN_ACCOUNTS_SWEEPER_SCHEDULE_HOURS`：自定义扫描小时（如 `0,6,12,18`，配置后优先）。
- `OPEN_ACCOUNTS_SWEEPER_SCHEDULE_MINUTE`：自定义扫描分钟（默认 `0`）。
- `OPEN_ACCOUNTS_SWEEPER_SECURITY_CHECK_ENABLED`：是否启用账号安全检测（默认开启）。
- `OPEN_ACCOUNTS_SWEEPER_CREATED_WITHIN_DAYS`：仅扫描最近 N 天创建的开放账号（`0` 不限制）。

## 6) 验证

- 访问：`/redeem/open-accounts`
- 完成 Linux DO 授权
- 按页面提示完成下单/上车流程
- 在管理后台查看 Credit 订单：`/admin/credit-orders`

## 7) LDC 商品小店（新增）

开放账号页已支持 `LDC 商品小店`，用于售卖可文本交付的商品（例如卡密、账号文本、密钥、卡片资料等）。

### 7.1 商品模型

后台「系统设置 -> 支付商品管理」新增了两个关键字段：

- `category`
  - `code`：兑换码商品（原有逻辑）
  - `ldc_shop`：LDC 小店商品（新增）
- `deliveryMode`
  - `inline`：购买后在页面展示交付内容
  - `email`：购买后仅邮件发送
  - `both`：页面展示 + 邮件发送

> `ldc_shop` 商品不依赖 `codeChannels`，库存来自“条目池”。

### 7.2 条目池（库存）管理

当商品 `category=ldc_shop` 时，可在商品列表点击「条目」管理库存：

- 每条库存是一个可售卖条目（`purchase_product_items`）
- 状态流转：`available -> reserved -> sold`（或手动 `offline`）
- 系统会在支付超时/失败时自动释放 `reserved` 条目，避免库存泄漏

### 7.3 订单与交付闭环

- 下单入口：`/redeem/open-accounts` 页面中的「LDC 商品小店」
- 支付通道：沿用 Linux DO Credit（`scene=ldc_shop_purchase`）
- 订单留痕：`ldc_shop_orders` 记录
  - 商品快照（名称/价格/交付方式）
  - 关联条目（item id + preview）
  - 支付状态、交付状态、错误信息
  - 页面交付时间、邮件发送时间

交付完成判定：

- `inline`：页面内容写入即完成
- `email`：邮件发送成功即完成
- `both`：页面写入 + 邮件发送都成功才标记完成

### 7.4 相关环境变量

- `LDC_SHOP_EMAIL_SUBJECT`：LDC 小店交付邮件标题（默认 `LDC 商品交付通知`）
- `LDC_SHOP_ORDER_QUERY_MIN_INTERVAL_MS`：前端轮询订单时，后端触发网关查单的最小间隔（默认 `8000`）
