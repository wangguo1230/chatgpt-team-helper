# 任务：支付宝口令订单商品化（独立商品菜单 + GPT单号/母号交付）

- 创建时间：2026-03-24 16:40:25
- 当前模式：执行
- 目标：在兼容历史支付宝口令订单接口的前提下，新增可配置商品与两类交付链路（GPT单号、GPT母号），并完成管理端、下单、处理、退回回滚与测试。

## 已批准执行范围
1. 复用现有表，按最小改造新增字段与子表。
2. 支付方式先支持支付宝口令，预留易支付扩展字段。
3. GPT单号：一个口令可购买多个数量；管理员处理后按数量消耗兑换码并向多个邮箱发送邀请。
4. GPT母号：一个口令可购买多个数量母号；按“兑换码数量 = 4”筛选可分配母号；处理后置不开放并记录订单占用关系。
5. GPT母号交付：页面可查订单状态；账号凭据仅邮件发送。
6. 账号管理新增 `gpt_password` 与 `email_password`，加密存储，仅管理员可查看明文。
7. 保持历史支付宝口令下单接口兼容（旧前端可继续工作）。
8. 订单失败/退款/退回回滚：未发凭据可恢复母号开放；已发凭据保持不开放并删除母号兑换码。
9. 覆盖数据库迁移、后端接口、后台页面、下单与交付、单元测试。

## 执行步骤
1. 数据层：扩展 `purchase_products`、`alipay_redpack_orders`、`gpt_accounts` 字段；新增母号占用子表与必要索引。
2. 安全层：新增密码字段加解密工具，接入 `gpt-accounts` 路由的写入与管理员明文读取权限控制。
3. 商品管理：新增“支付宝口令商品管理”独立菜单、后端管理接口与独立前端页面（不复用旧设置页商品区）。
4. 下单链路：扩展支付宝口令下单与订单详情模型，支持商品类型、购买数量、多邮箱输入（单号）与邮箱收件地址（母号）。
5. 管理处理链路：
   - GPT单号：处理时按数量逐个邀请邮箱并消耗兑换码；记录批量处理结果。
   - GPT母号：处理时按数量分配母号（`兑换码数量=4`），发送凭据邮件，标记母号不开放并记录占用。
6. 退回/退款：根据“是否已发送母号凭据”执行回滚策略；必要时恢复开放状态或执行兑换码清理。
7. 前端：
   - 用户侧支付宝口令下单页新增商品选择与类型感知输入。
   - 管理侧订单页扩展批量处理与母号分配/交付展示。
   - 新增独立“支付宝口令商品管理”菜单页。
8. 邮件模板：新增母号交付邮件模板与发送逻辑。
9. 测试：新增/更新后端单测，执行 `npm run test --workspace=backend` 并修复回归。

## 预期结果
- 历史接口可用，新增能力可配置、可下单、可处理、可退回。
- 明文密码仅管理员可见，数据库中为密文。
- 新增独立商品管理菜单页完成上线能力。
