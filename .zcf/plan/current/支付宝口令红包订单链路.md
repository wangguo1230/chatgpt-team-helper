# 支付宝口令红包订单链路

## 背景与目标
在现有订单管理体系内新增“支付宝口令红包”订单闭环，支持公开页提交/补录、后台快速邀请、状态同步、管理员邮件通知、账号同步自查联动，并与兑换码强关联。

## 核心业务规则
1. 状态机：`pending -> invited -> redeemed`。
2. 公开页字段：邮箱、支付宝口令、备注（可选），无需手机号。
3. 支付宝口令全局唯一；同邮箱不同口令视为多订单。
4. 补录不允许跨邮箱改绑既有口令订单。
5. 创建/补录通知管理员邮箱（按汇总发送）。
6. 快速邀请后需支持账号管理“同步自查”。

## 兑换码关联要求（本次增补）
1. 公开创建订单时，自动在 `redemption_codes` 创建/关联 `channel=alipay_redpack` 兑换码。
2. 订单列表展示兑换码关联信息（ID、消耗时间）。
3. 快速邀请前先消耗订单关联兑换码；若邀请未发出且本次刚消耗，回滚兑换码。
4. 快速邀请成功后，订单保留兑换码关联与消耗记录。

## 公开页补充要求（本次增补）
1. 增加库存展示（可邀请库存）。
2. 提供库存刷新能力。

## 执行步骤
1. 数据库：`alipay_redpack_orders` 增加兑换码关联字段及索引。
2. 服务层：订单服务实现创建即关联兑换码、消耗与回滚能力。
3. 快捷邀请：支持 `consumeCode=false`，与订单兑换码消耗分离。
4. 路由层：补齐公开接口、后台订单接口、快速邀请及状态同步。
5. 前端：公开页/后台页接入，后台展示兑换码记录，公开页展示库存。
6. 文档：输出从注册配置到使用的全链路说明与排障指引。
7. 验证：语法检查、后端测试、前端类型检查、关键链路回归。
