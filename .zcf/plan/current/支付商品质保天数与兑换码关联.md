# 支付商品质保天数与兑换码关联

## 目标
在现有支付商品体系上，确保商品与兑换码按 `order_type + service_days` 关联，公开页展示商品库存，订单展示商品与兑换码关联信息，并统一前端中文文案显示（值不变）。

## 已执行
1. 后端：`purchase` 链路库存统计与预留兑换码按商品 `orderType/serviceDays` 匹配。
2. 后端：支付订单后台列表接口增加 `productKey/codeId/code/codeAccountEmail/codeChannel` 返回。
3. 前端：支付订单管理页增加“关联兑换码”展示，并显示商品键、质保类型、质保天数。
4. 前端：商品管理页将 `warranty/no_warranty/anti_ban` 显示改为中文标签（值保持不变）。
5. 验证：后端测试与前端类型检查通过。

## 说明
